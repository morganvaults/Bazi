# 八字命理 Skill v2

用于八字排盘、四柱结构分析、综合评分、用神喜忌、大运流年、事业财运、婚恋关系、五行调补建议。

## 当前能力

- 本地排盘：公历生日、时辰、性别输入，输出四柱、十神、五行、强弱、格局、用神、大运和评分。
- 四柱分析：直接输入四柱时输出原局结构评分、墓库、空亡、十二长生、合化、清浊和刑冲合害。
- 运年分析：完整生日可加 `--liu-nian` 或 `--yun-years` 分析大运、流年与原局交互。
- 问真校准：显式 `--wenzhen` 时调用问真八字公开 API 反查候选生日或校准起运、大运；默认本地运行不联网。
- 兼容输出：保留旧 JSON 字段，并新增 `评分系统`、`墓库`、`空亡`、`十二长生`、`合化成败`、`格局清浊`、`大运流年交互`。

## 先判断资料等级

- **S 级**：完整公历出生日期、时间、性别、出生地，且用户明确是否要真太阳时。可做完整排盘、大运、流年、评分。
- **A 级**：公历日期、小时、性别。可做标准排盘，默认本地标准时，不启用真太阳时。
- **B 级**：只给四柱八字。先做本地“四柱结构分析/结构评分”。若用户需要反查出生时间、校准起运或分析精确大运流年，再使用问真 `--wenzhen --pillars` 输出候选；候选确定后用 `--candidate-index` 校准四柱、起运、交运和大运。
- **C 级**：只问“算八字/看运势”但无资料。先索取出生信息。

## 推荐工具

当前脚本为纯 Python 内置实现，无第三方依赖。默认使用本地算法，不联网。只有显式传入 `--wenzhen` 时才调用问真八字公开接口。支持 1900-2099 的农历转换基础数据；节气/月令按内置节气规则处理。若用户要求天文级分钟精度，可用 `--wenzhen` 复核起运，或提示需要专业万年历/精确节气表复核。

## 核心方法

1. **历法与月令**
  - 年柱以立春为界，不以农历正月初一或公历 1 月 1 日为界。
  - 月柱以节气为界：立春寅、惊蛰卯、清明辰、立夏巳、芒种午、小暑未、立秋申、白露酉、寒露戌、立冬亥、大雪子、小寒丑。
  - 日柱用儒略日换算；时柱用五鼠遁。
2. **五行统计**
  - 同时输出简单五行和加权五行。
  - 加权五行：天干 1.0；地支藏干主气 1.0、中气 0.6、余气 0.3；月令额外加权。
3. **日主强弱**
  - 先看月令，再看通根、透干、生扶克泄耗、合冲刑害。
  - 输出身弱、偏弱、中和、偏旺、身旺、极旺、从弱候选、从旺候选。
4. **用神喜忌**
  - 第一层：扶抑用神。身强取泄耗克，身弱取印比。
  - 第二层：调候用神。夏月燥热取水金，冬月寒湿取火木。
  - 第三层：通关用神。五行交战时取通关，不机械套固定表。
5. **大运**
  - 阳男阴女顺排，阴男阳女逆排。
  - 顺排看出生到下一节，逆排看出生到上一节。
  - 三天一岁，一天四个月，输出起运岁月和每步大运。
6. **评分**
  - 完整命盘评分与四柱结构评分分开说明。
  - 无出生日期时，神煞、大运、流年不做高精度判断。
  - 资料完整度与历法可信度只作为资料等级提示，不计入综合评分。
  - `原局结构分` 看先天结构顺畅度；`领域评分` 看不同主题；`运势兑现评分` 看指定大运流年对原局的激活。
  - 评分是命理结构参考，不作医学、法律、投资结论。
7. **墓库、空亡、十二长生**
  - 墓库分析分层判断四墓库支、日主墓、五行库、十神库，不把所有辰戌丑未都当作日主入墓。
  - 空亡分析用于判断旬空落在何柱，提示虚、迟、待填实的结构。
  - 十二长生用于辅助判断日主在四支的生旺墓绝，不单独决定格局高低。
8. **合化、清浊、运年交互**
  - 合化成败看化神是否得令、透干、有根、受助、被冲破，以及是否为喜用。
  - 格局清浊看月令格神、喜忌透藏、官杀/食伤/印枭混杂、刑冲破局与合化助忌。
  - 大运流年交互需要完整生日和性别；只给四柱时默认不做精确起运和流年交互。若用户需要精确运年，先用问真反查候选，取得性别与确定候选后再校准起运、大运和流年交互。

详细规则见 `references/methodology.md`。

## 输出风格

默认文本输出使用 `summary` 短版，适合日常看盘与快速问答。若用户要求“详细排盘、完整技术校验、全部大运、墓库空亡十二长生细节”，使用 `--detail full`。若用户希望比默认略详细但不要完整长文，使用 `--detail standard`。普通文本默认按 `--wrap-width 72` 主动换行，长大运、评分细项、墓库/空亡说明和刑冲合害说明会保留缩进后折行；`--json` 和 `--score-only` 不做 word wrap。

`summary` 默认结构：

1. 资料等级与是否可精排
2. 四柱、日主强弱、格局、综合评分
3. 五行与用神喜忌
4. 墓库、空亡、十二长生关键结构
5. 合化成败、格局清浊重点
6. 刑冲合害重点
7. 当前大运与下一步大运；指定流年时显示 `年份（岁数）` 和运年交互重点

`full` 完整结构保留：资料等级、四柱、十神、五行、日主强弱、格局、用神喜忌、墓库、空亡、十二长生、合化成败、格局清浊、综合评分细项、刑冲合害、完整大运、大运流年交互。

语气要稳、清楚、少绝对化。使用“倾向、较容易、需要注意”，避免“必定、绝对、一定破败”等恐吓式断语。
