# bazi-v2 更新内容

## v2.5.1 文档与发布整理

- 缩短 `SKILL.md` frontmatter description，使触发条件更清楚且不把完整流程塞进元数据。
- 明确默认本地运行不联网；只有显式传入 `--wenzhen` 才调用问真八字公开 API。
- 整理四柱输入流程：默认先做本地结构分析，需要反查候选或校准起运时再走问真。
- 增加可运行验证样例，方便发布前检查。

## v2.5.0 问真八字 API 接入

- 新增 `--wenzhen`：显式调用问真八字公开接口，默认本地排盘不联网。
- 新增 `--candidate-index`：四柱反查候选出生时间后选择具体候选。
- 四柱输入流程：`--wenzhen --pillars` 输出候选；选择候选后调用问真排盘，返回起运、交运和大运。
- 公历生日流程：`--date/--time/--gender --wenzhen` 直接用问真接口校准起运和大运。
- JSON 输出新增 `问真八字` 字段，文本输出的大运部分使用问真起运信息。

## v2.1 输出层级

- 新增 `--detail {summary,standard,full}`。
- 默认 `summary`：短输出，日常看盘优先展示四柱、强弱、格局、评分、用神、关键结构和前两步大运。
- `standard`：中等输出，保留主要模块并压缩墓库、空亡、十二长生和大运。
- `full`：完整输出，保留 v2 原有长文本结构、完整评分细项和完整大运。
- 新增 `--wrap-width`，普通文本默认按 72 字符左右主动换行；设为 `0` 可关闭。
- `--json` 与 `--score-only` 不变，继续兼容旧调用。

## v2.2 技法增强

- 新增 `合化成败` JSON/text 模块。
  - 覆盖天干五合、地支六合、三合局、半合/拱合。
  - 输出成化分、状态、化神和成败理由。
- 新增 `格局清浊` JSON/text 模块。
  - 输出清气分、清浊等级、清气来源、浊气来源、成格阻点和取清建议。
  - `综合评分 -> 格局清纯度与成格程度` 参考清气分，但字段名保持兼容。
- 新增 `大运流年交互` JSON/text 模块。
  - `--liu-nian 2026` 分析单年。
  - `--yun-years 2026-2035` 分析区间。
  - `--focus career|wealth|marriage|health|general` 控制主题重点。
- 大运每步新增起始日期、结束日期、起始年份、结束年份、顺逆来源。

## v2.2.1 输出与评分调整

- `资料完整度与历法可信度` 不再参与综合评分，也不再作为评分分项输出。
- 综合评分按剩余结构分项从 90 分归一到 100 分；JSON 保留 `原始分项` 便于核对。
- 大运流年交互文本输出使用 `年份（岁数）`，例如 `2026（20）丙午`。
- 普通文本去掉冗余总结/说明行，输出更短但保留核心触发点。

## v2.3 评分系统拆分

- 新增 JSON 字段 `评分系统`。
- `原局结构分`：沿用结构评分，代表原局顺畅度。
- `领域评分`：新增事业、财运、婚姻、健康倾向、变动适配度。
- `运势兑现评分`：指定 `--liu-nian` 或 `--yun-years` 时输出当前/区间/逐年兑现分。
- 旧字段 `综合评分` 保留兼容，但文本输出改称“原局结构”。

## v2.4 墓库分层

- 墓库输出拆分为四层：四墓库支、日主墓、五行库、十神库。
- 日主墓按十二长生判断，避免把所有辰戌丑未都误称为“墓”。
- 新增 `联动解析`：结合空亡、十二长生、透干、冲库、合化和运年引动。
- 大运流年交互中，若运年引动原局墓库，会在提示中标注对应地支与十神库。
- 变动适配度改为参考四墓库支数量与日主墓数量。

## v2.4.1 字段命名调整

- `动荡指数` 改名为 `变动适配度`。
- 文本输出只展示 `变动适配度`。
- JSON 保留 `动荡指数` 作为旧字段兼容别名。

## 新增模块

### 墓库

- 识别辰、戌、丑、未四墓库。
- 标注五行库：辰为水库，戌为火库，丑为金库，未为木库。
- 按日主转换为十神库：财星库、官杀库、印星库、食伤库、比劫库。
- 判断库中五行是否透干。
- 判断辰戌冲、丑未冲是否冲开墓库。
- 对墓库偏多、藏而待引、冲开可动给出综合提示。

### 空亡

- 按日柱所在旬计算旬空。
- 输出原局四支是否落空。
- 标注落空所在柱位与对应十神。
- 空亡只作虚、迟、待填实提示，不单独定吉凶。

### 十二长生

- 按日干计算四支十二长生状态。
- 支持长生、沐浴、冠带、临官、帝旺、衰、病、死、墓、绝、胎、养。
- 输出每柱阶段和力量提示。
- 十二长生用于辅助旺衰与墓库判断，不替代月令。

## 排盘校准

- 修正日柱儒略日锚点：避免旧版日柱整体偏移。
- 年柱/月柱在测试覆盖年份支持按节令具体时刻切换，不只按节气日期 00:00 切换。
- 大运起运继续按“节”计算，不取中气。

## 兼容性

- 原有 `四柱`、`藏干`、`十神`、`五行`、`日主强弱`、`格局`、`用神喜忌`、`刑冲合害`、`综合评分`、`大运`、`农历` 字段保持不变。
- 新增 JSON 字段：`墓库`、`空亡`、`十二长生`。
- 原有 `--pillars`、`--date`、`--time`、`--gender`、`--score-only`、`--json` 参数保持可用。
- 新增 `--detail` 只影响普通文本输出，不影响 JSON 与 score-only。
- 新增 `--wrap-width` 只影响普通文本输出，不影响 JSON 与 score-only。
- 新增 `--liu-nian`、`--yun-years`、`--focus` 只在完整生日排盘时提供精确大运流年交互；只给四柱时仍输出原局合化与清浊。
- 新增 JSON 字段：`合化成败`、`格局清浊`、`大运流年交互`。
